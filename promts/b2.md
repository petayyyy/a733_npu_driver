TASK B2-chat-shell: Build a clean, usable chat wrapper so a human can load a model and chat with
the LLM running NPU-only on the Orange Pi Zero 3W, interactively. Prefer Python for the user-
facing shell (calling the existing C runner), unless a thin C++ CLI is clearly simpler.

REQUIREMENTS:
1. Interactive REPL: user types a message, the model streams its reply token-by-token to the
   terminal (show tokens as they are generated, not all-at-once), then prompts for the next turn.
2. Loads the NBG ONCE at startup (reuse the T10b persistent runner: load network once, submit
   token windows in a loop -- do NOT relaunch per token).
3. Uses the model's real HF tokenizer (CPU side) for encode/decode. Apply the model's chat
   template (e.g. SmolLM2 ChatML-style) so replies are coherent in a chat setting.
4. Fixed-window handling made explicit and honest: the window W is fixed (e.g. 32/64). Implement
   the sliding window correctly -- when prompt+generation exceeds W, slide and clearly tell the
   user context is limited to the last W tokens. Show a live token counter (used / W).
5. CLI flags: --model PATH (NBG), --tokenizer PATH, --window N, --max-new-tokens N,
   --temperature / greedy. Print startup info: NBG size, device (/dev/vipcore), tok/s after each
   reply, and a one-line "NPU-only: model layers on NPU, CPU does tokenize/argmax/loop" notice.
6. Graceful stop (Ctrl-C ends the turn cleanly), and a /reset command to clear the window.

ARCHITECTURE: the C runner does the NPU forward + argmax/sampling and exposes a simple line-
protocol over stdin/stdout (or a small local socket) so the Python shell can stream tokens
without reloading. Keep model-layer compute on the NPU; CPU only does I/O, tokenization, the
sliding window, sampling, and the UI.

DELIVERABLE: scripts/board/chat_shell.py (+ any runner protocol changes), a README section with
exact run commands on the Orange Pi, and reports/b2-chat-shell.md showing a real multi-turn
session transcript with live tok/s. Validate on the working SmolLM2-135M int16 NBG.

SUCCESS GATE: a human can run one command on the Orange Pi, chat for several turns with streamed
NPU-generated replies, see tok/s and the window counter, and the model stays coherent within the
window. Committed.

START FROM: the T10b persistent runner (scripts/board/npu_lm_runner.c, build-npu-lm-runner.sh)
and board VIPLite layout; the working SmolLM2-135M int16 NBG and its HF tokenizer.
