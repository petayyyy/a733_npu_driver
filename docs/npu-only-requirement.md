# NPU-Only Requirement

Date: 2026-06-20

This document is the active project constraint and supersedes earlier hybrid
CPU-decoder recommendations in `task.md` and prior reports.

## Hard Requirement

LLM and VLM inference must execute on the A733 Vivante VIP9000 NPU.

CPU execution is allowed only for orchestration and non-inference support work,
including:

- file I/O and model loading;
- tokenization and detokenization until an NPU-side replacement exists;
- image decode and formatting until an NPU-side replacement exists;
- launching VIPLite/awnn/vpm runtime calls;
- debug logging, validation, and comparison tooling.

The CPU must not be the primary compute engine for LLM/VLM model layers.

## Success Criteria

An LLM/VLM milestone counts only when the model compute graph runs on the NPU.
For language models this means, at minimum:

- embeddings or equivalent token input handling feeding an NPU graph;
- transformer block compute on NPU, including attention and MLP subgraphs;
- logits generated by the NPU graph;
- fixed-shape sequence handling documented if dynamic KV-cache is unavailable.

For VLMs this means:

- vision encoder on NPU;
- projector/adapter on NPU, if present;
- language decoder or decoder block on NPU;
- end-to-end image-to-text path where model-layer compute is not delegated to
  CPU.

## Historical Baselines

CPU decoder work, including the llama.cpp report, is retained only as a
diagnostic baseline and does not satisfy the project goal. It can be useful for
checking prompts, tokenization, or expected text behavior, but it is not a
deliverable for this project.
