# G3a Tiny VLM Bridge NPU Validation

Date: 2026-06-20

## Purpose

Validate a tiny fixed-shape VLM bridge where image-prefix projection, token
embedding lookup, text/vision concatenation, decoder compute, and logits
generation all execute inside an A733 NBG on the VIP9000 NPU.

This is not a CPU VLM or CPU decoder path. CPU-side work in this test is
limited to preparing raw input files, launching `vpm_run`, and comparing
outputs. The `image_embed` input represents the output contract of the
previously validated MobileCLIP-S0 NPU vision encoder.

## Model

Generated by:

```bash
python3 scripts/host/make_tiny_vlm_bridge_onnx.py \
  --output-dir work/generated/tiny_vlm_bridge
```

Topology:

- Input 0: `image_embed`, shape `1x512`, dtype `float32`.
- Input 1: `token_ids`, shape `1x4`, dtype `int32`.
- Tokens used for validation: `1 5 9 2`.
- Image projector/adapter: `MatMul + Add`, mapping `1x512` to one `1x1x8`
  image prefix token.
- Text path: ONNX `Gather` from a learned `16x8` token embedding table, plus
  learned position embedding.
- VLM bridge: `Concat` image prefix and four text tokens into `1x5x8`.
- Decoder block: LayerNorm patterns, causal self-attention, MLP with GELU,
  residual connections, final LayerNorm.
- Output: `logits`, shape `1x5x16`.

ONNX ops:

- `Gather`
- `MatMul`
- `Transpose`
- `Concat`
- `Reshape`
- `Add`, `Sub`, `Mul`, `Div`
- `ReduceMean`, `Sqrt`
- `Softmax`
- `Erf`

This extends the tiny LM probe by adding the NPU-side VLM projector/adapter and
image-prefix fusion path.

Import, quantization, host inference, and NBG export completed with
`Error(0),Warning(0)`. ACUITY matched the image projector as a `fullconnect`
layer and the token embedding as a native `gather` layer before the decoder
block.

Packaging note: the board `vpm_run` parser requires LF-only `sample.txt`.
Windows CRLF makes the first token parse as `[network]\r` and fails with
`Bad task file. Wrong line @ 0`.

## Result

Gate G3a tiny-VLM-bridge subgate passed: a static VLM bridge with image
projector/adapter, token embedding `Gather`, image/text concatenation, causal
attention, MLP, LayerNorm-style reductions, and logits projection was compiled
to an A733 NBG and validated on the VIP9000 NPU.

This completes the first NPU-only connection between the previously validated
MobileCLIP-S0 encoder output contract and the NPU language path. The next work
is scaling the fixed-shape language/VLM path into a decode loop where CPU only
updates token IDs, moves tensors between NPU graph stages if needed, and
postprocesses logits, while every model-layer evaluation remains on NPU.
