# G3a Tiny Decoder Block NPU Validation

Date: 2026-06-20

## Purpose

Validate the first NPU-only transformer decoder subgraph on the A733 VIP9000.
This is not a CPU decoder path. The board runs a static NBG containing decoder
model-layer compute and emits logits from the NPU graph.

The probe is intentionally tiny so the first gate tests operator/toolchain
compatibility rather than model quality.

## Model

Generated by `scripts/host/make_tiny_decoder_block_onnx.py`.

Topology:

- Input: `hidden_in`, shape `1x4x8`, float embedding tensor.
- Decoder block: LayerNorm patterns, causal self-attention, MLP with GELU,
  residual connections, final LayerNorm.
- Output: `logits`, shape `1x4x16`.
- ONNX ops: MatMul, Transpose, Add, Sub, Mul, ReduceMean, Sqrt, Div, Softmax,
  Erf.

The graph starts from an embedding tensor instead of token IDs. That keeps the
probe focused on transformer block compute and logits generation on NPU. Token
embedding lookup is part of the next NPU language-model gate.

- ACUITY Docker image: `ubuntu-npu:v2.0.10.1`

## Result

Gate G3a decoder-block subgate passed: a transformer decoder block with
attention, Softmax, GELU, LayerNorm-style reductions, residuals, and logits
projection was compiled to an A733 NBG and validated on the VIP9000 NPU.

This does not yet complete the LLM target. The next gate is a tiny fixed-shape
language model where token embeddings, decoder compute, and logits are all in
the NPU graph.
