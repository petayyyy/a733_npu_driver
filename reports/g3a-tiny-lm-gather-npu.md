# G3a Tiny LM Gather NPU Validation

Date: 2026-06-20

## Purpose

Validate a tiny fixed-shape language-model graph where token input handling,
embedding lookup, decoder compute, and logits generation all execute inside an
A733 NBG on the VIP9000 NPU.

This is not a CPU decoder path. CPU-side work in this test is limited to
preparing the raw token-id input file, launching `vpm_run`, and comparing
outputs.

## Model

Generated by:

```bash
python3 scripts/host/make_tiny_lm_onnx.py \
  --variant gather \
  --output-dir work/generated/tiny_lm_gather
```

Topology:

- Input: `token_ids`, shape `1x4`, dtype `int32`.
- Tokens used for validation: `1 5 9 2`.
- Embedding: ONNX `Gather` from a learned `16x8` token embedding table.
- Position embedding: learned `1x4x8` tensor added inside the graph.
- Decoder block: LayerNorm patterns, causal self-attention, MLP with GELU,
  residual connections, final LayerNorm.
- Output: `logits`, shape `1x4x16`.

ONNX ops:

- `Gather`
- `MatMul`
- `Transpose`
- `Add`, `Sub`, `Mul`, `Div`
- `ReduceMean`, `Sqrt`
- `Softmax`
- `Erf`

This extends the previous decoder-block probe by moving token embedding lookup
inside the NPU graph.

Host toolchain:

- ACUITY Docker image: `ubuntu-npu:v2.0.10.1`
- NPU optimize target: `VIP9000NANODI_PLUS_PID0X1000003B`

## Result

Gate G3a tiny-language-model subgate passed: a static token-id language model
with ONNX `Gather` token embeddings, position embeddings, causal attention, MLP,
LayerNorm-style reductions, and logits projection was compiled to an A733 NBG
and validated on the VIP9000 NPU.

The next NPU-only work is to connect this pattern into the VLM path:
MobileCLIP-S0 encoder output, NPU projector/adapter, and an NPU language decoder
graph. Scaling the language path also needs a fixed-window decode loop where CPU
only updates token IDs and performs postprocessing, while every model-layer
evaluation remains on NPU.
