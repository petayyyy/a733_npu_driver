# Gate G2 Report - ACUITY Inception v1 ONNX Conversion

Date: 2026-06-20 local / 2026-06-19 UTC

## Source

Radxa ACUITY setup reference:

- https://docs.radxa.com/en/cubie/a7z/app-dev/npu-dev/cubie-acuity-env

The same A733 flow used for LeNet was reused here:

- Docker image: `ubuntu-npu:v2.0.10.1`
- SDK: `https://github.com/ZIFENG278/ai-sdk.git`
- A733 build setting: `AI_SDK_PLATFORM=a733`
- ACUITY NPU target: `NPU_VERSION=v3`

## Model

- Model: Inception v1 from `ZIFENG278/ai-sdk/models/inception_v1`
- Original framework: ONNX
- Input: `1x3x224x224`, NCHW
- Calibration/image input: `space_shuttle_224x224.jpg`
- Preprocess from `inception_v1_inputmeta.yml`:
  RGB input, reverse channel, mean `104 117 124`, scale `0.00390625`
- Target optimization: `VIP9000NANODI_PLUS_PID0X1000003B`

## ACUITY Commands

```bash
export ACUITY_PATH=/root/acuity-toolkit-whl-6.30.22/bin
export VIV_SDK=/root/Vivante_IDE/VivanteIDE5.11.0/cmdtools
source env.sh v3
pegasus_one inception_v1
../scripts/pegasus_quantize.sh inception_v1 int16
../scripts/pegasus_inference.sh inception_v1 int16
../scripts/pegasus_export_ovx.sh inception_v1 int16
```

The uint8 path was generated by `pegasus_one inception_v1`; int16 was generated
with the explicit quantize/inference/export commands above. Both paths completed
with ACUITY export `Error(0),Warning(0)`.

## Baseline Comparison

ACUITY's non-quantized inference output for the same image has this top-5:

```text
885: 0.163503855
911: 0.030567363
904: 0.029327698
794: 0.029267265
700: 0.021456530
```

Quantized host-side ACUITY inference and board-side NPU output were compared
using the dumped 1000-class tensors:

| Comparison | Top-1 matches | Top-5 set matches | Max abs diff | Mean abs diff |
|---|---:|---:|---:|---:|
| ACUITY uint8 vs NPU uint8 | yes | yes | 0.004674390 | 0.000015351 |
| ACUITY int16 vs NPU int16 | yes | yes | 0.000063613 | 0.000005457 |
| ONNX/non-quantized vs NPU uint8 | yes | no | 0.013601512 | 0.000058940 |
| ONNX/non-quantized vs NPU int16 | yes | yes | 0.000060455 | 0.000005530 |

The int16 path preserves the non-quantized top-5 order and scores closely. The
uint8 path preserves top-1 but shifts the lower top-5 ranks and confidence, so
int16 remains the preferred G2/Phase 3a quantization mode.

## Gate Result

G2 is passed for an ONNX CNN beyond the toy LeNet sample. ACUITY converts
Inception v1 to A733-compatible NBG files for both uint8 and int16; both run on
the Radxa Cubie A7Z VIP9000 through `/dev/vipcore`, identify the expected
`cid=0x1000003b`, and return successful inference through `vpm_run`.
